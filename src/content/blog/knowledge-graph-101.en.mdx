## Keywords find files; graphs find context

Search returns *documents*. A **knowledge graph** returns *how ideas connect* — who owns a decision, which project blocked another, which policy applies to a customer segment.

## When graphs help most

- Cross-functional incidents (“why did we ship X?”)
- Compliance and policy Q&A
- Onboarding (“what should I read first?”)

The `/app/graph` view visualizes indexed relationships. Q&A uses the same store so answers stay grounded.
